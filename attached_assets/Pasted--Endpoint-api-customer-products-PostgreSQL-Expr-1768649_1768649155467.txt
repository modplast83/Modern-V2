عنوان المشكلة

بطء واضح في تحميل الواجهة بسبب بطء Endpoint /api/customer-products (PostgreSQL + Express)

وصف المشكلة

عندي مشروع Node/Express يعمل على Replit مع Vite + React. التطبيق يعمل بشكل طبيعي بدون أخطاء، لكن يوجد بطء ملحوظ في الواجهة بسبب استدعاء API واحد بالتحديد.

في السجلات يظهر بشكل متكرر:

GET /api/customer-products took 660ms

GET /api/customer-products 304 in 720ms :: {"data":"[Array:4818]"}

وأحياناً: took 533ms / 593ms

الـ endpoint يرجع حوالي 4818 سجل كل مرة، وهذا يسبب تأخير في صفحة الطلبات/الطباعة.

البيئة (Environment)

NODE_ENV=development

Server: tsx server/index.ts

Express API على port 5000

قاعدة البيانات: PostgreSQL

عدد السجلات الكبيرة:

customers: حوالي 3173

customer-products: حوالي 4818

المطلوب منكم (Request)

أحتاج مساعدتكم في تحسين الأداء وتقليل زمن الاستجابة للـ endpoint:

GET /api/customer-products

الهدف: تقليل الاستجابة من ~600–720ms إلى زمن أقل قدر الإمكان.

نقاط تقنية مرجّحة لسبب البطء

الاستعلام يرجع بيانات كبيرة جداً (4818 صف) كل مرة.

احتمال عدم وجود Index مناسب على الأعمدة المستخدمة في الاستعلام/الفلترة.

احتمال بطء نقل البيانات أو عدم وجود ضغط gzip/br.

احتمالية أن الاستعلام يقرأ من جدول كبير بدون pagination أو filtering.

اقتراحات للتحسين (مطلوب تطبيق واحد أو أكثر)
1) إضافة Filtering بدل جلب كل البيانات

نحتاج دعم query مثل:

/api/customer-products?customer_id=...

أو /api/customer-products?ids=...

بدل جلب جميع الـ 4818 سجل.

2) تفعيل Pagination

مثلاً:

?page=1&limit=200

3) إضافة Indexes على PostgreSQL

يرجى التأكد من وجود Index على الأقل على:

customer_products(customer_id)

و/أو أي عمود يتم الفلترة به

production_orders(order_id) (إذا كانت تستخدم في joins)

PK على customer_products(id) موجود غالباً

4) تفعيل ضغط الاستجابة على Express

استخدام middleware مثل:

compression()

لتقليل حجم الـ payload المنقول.

5) Cache مؤقت (اختياري)

إذا البيانات لا تتغير كثيراً، ممكن تطبيق caching بسيط لمدة 30-60 ثانية.
